# Nexu 桌面端：官方 vs 自建 完全对比

## 🎯 核心结论

**好消息：官方桌面端就是开源版！两者是同一个东西。**

Nexu 采用 MIT 开源协议，官方发布的桌面端就是从 GitHub 仓库构建出来的。你：
- ✅ 可以直接用官方发布的安装包（最简单）
- ✅ 也可以自己从源码构建（完全免费，可二次开发）

---

## 📦 方案一：直接用官方桌面端（推荐）

### 为什么推荐？
- **开箱即用**：双击安装，无需任何技术背景
- **稳定可靠**：官方维护，定期更新
- **完全免费**：BYOK 模式（自带API Key），不收订阅费
- **数据本地**：所有对话、配置都在你电脑上

### 获取方式
1. 访问官网下载：[https://nexu.io](https://nexu.io)
2. Windows: 下载 `.exe` 安装包
3. Mac: 下载 `.dmg` 安装包
4. 双击安装 → 配置 API 地址 → 扫码接入微信/飞书

---

## 🔧 方案二：自己从源码构建（高级用户）

### 什么时候需要自建？
- 需要深度定制界面或功能
- 想加自己品牌的Logo/名称
- 需要集成特殊功能（如对接 TripClaw 的定制接口）
- 想发布自己的版本给客户用

### 构建步骤
```bash
# 1. 克隆仓库
git clone https://github.com/nexu-io/nexu.git
cd nexu

# 2. 安装依赖
pnpm install

# 3. 开发模式运行（热重载）
pnpm run dev:desktop

# 4. 生产构建
pnpm run build

# 5. 构建产物在
apps/desktop/dist/
```

### 系统要求
- Node.js 22+
- pnpm 10+
