## 🎯 技能概述

### 什么是技能（Skills）？

技能是预编译的知识模块，通过**运行时注入**的方式增强 Agent 的能力：

```
传统模式：
  用户请求 → Agent → LLM → 基础回复

技能增强模式：
  用户请求 → Agent → [注入技能知识] → LLM → 专业回复
                       ↑
                    运行时动态注入
                    不修改源代码
```

### 技能的核心优势

| 特性 | 说明 |
|------|------|
| **零代码修改** | 不修改 `router.js`、`agents/*.js`、`index.js` |
| **Git 仓库干净** | 技能独立于项目代码，不影响版本控制 |
| **即插即用** | 安装后立即可用，无需重启（部分技能） |
| **随时回滚** | 卸载技能，系统自动恢复基础功能 |
| **可组合性** | 多个技能可协同工作，增强同一 Agent |

## ⚙️ 技能作用机制

### 1. 知识注入原理

技能通过修改 LLM 的**系统提示词（System Prompt）**来增强 Agent：

```javascript
// 伪代码示例
const basePrompt = `我是小蓝，旅业云AI的行程规划师 🗺️`;

// 技能注入
const enhancedPrompt = `
${basePrompt}

【⚡ 技能增强知识（来自 travel-planner）】
- 全球300+城市深度数据库
- 实时景点开放状态查询
- 历史文化背景自动关联
- 亲子/情侣/老年专项路线模板
`;

await llm.chat(enhancedPrompt, history, message);
```

### 2. 技能与代码的隔离

| 层级 | 技能位置 | 是否修改代码 | 是否进入Git |
|------|---------|-----------|------------|
| **源代码层** | `C:\WorkBuddy\Claw\openclaw-server\` | ❌ 否 | ✅ 是 |
| **技能层** | `C:\Users\Administrator\.agents\skills\` | ❌ 否 | ❌ 否（.gitignore） |
| **运行时层** | 内存（临时注入） | ❌ 否 | ❌ 否 |

### 3. 技能加载流程

```
1. 用户发送请求 → TripClaw API
2. router.js 路由到对应 Agent（如 xiaolan）
3. Agent 加载基础 SOUL（agents/xiaolan.js）
4. 技能注入器检测已安装技能
5. 动态注入技能知识到 SOUL
6. 调用 LLM（带增强后的提示词）
7. 返回专业回复
```

## 🚧 故障排查

### 问题1：技能安装失败

**症状**：运行 `npx skills add ...` 报错

**排查步骤**：

1. 检查网络连接
```bash
ping github.com
```

2. 检查 Node.js 版本
```bash
node --version
# 推荐：v18+ 或 v20+
```

3. 清理缓存后重试
```bash
npm cache clean --force
npx skills add ailabs-393/ai-labs-claude-skills@travel-planner -g -y
```

**解决方案**：
- 如网络问题：使用代理或切换网络
- 如版本问题：升级 Node.js
- 如权限问题：使用 `sudo`（Linux/Mac）或以管理员身份运行（Windows）

---

### 问题2：技能未生效

**症状**：安装后，Agent 回复没有变化

**排查步骤**：

1. 确认技能已安装
```bash
npx skills list -g
```

2. 检查技能是否启用
```bash
npx skills status travel-planner
```

3. 查看服务日志
```bash
node index.js --verbose
```

**解决方案**：

- 如技能未启用：
```bash
npx skills enable travel-planner
```

- 如需要重启服务：
```bash
# 停止服务（Ctrl+C）
# 重新启动
node index.js
```

### 问题3：Git 仓库显示技能文件

**症状**：`git status` 显示技能文件被跟踪

**解决方案**：

1. 添加到 `.gitignore`
```bash
cat .gitignore
# 确保包含：
.agents/
~/.agents/
```

2. 移除已跟踪的文件
```bash
git rm -r --cached .agents/
```

3. 提交更改
```bash
git add .gitignore
git commit -m "fix: ignore skills directory"
```

---

### 问题4：技能导致响应变慢

**症状**：LLM 响应时间明显变长（>10秒）

**排查步骤**：

1. 测试单个技能的影响
```bash
# 禁用所有技能
npx skills disable-all

# 启用一个技能
npx skills enable travel-planner

# 测试响应时间
time curl -X POST http://localhost:3000/chat \
  -H "Content-Type: application/json" \
  -d '{"message":"测试","agent":"xiaolan"}'
```

**解决方案**：

- 临时禁用消耗大的技能
```bash
npx skills disable ljg-travel  # 12个Agent并行，可能较慢
```

- 优化 LLM 选择
  - 使用 Claude 3.5 Sonnet（最快）
  - 避免 GPT-4（较慢）

---

### 问题5：多个技能冲突

**症状**：两个技能对同一个 Agent 增强，结果混乱

**排查步骤**：

1. 查看 Agent 的所有增强技能
```bash
npx skills list --agent xiaolan
```

2. 对比单技能和双技能的效果
```bash
# 只启用 travel-planner
npx skills enable travel-planner
npx skills disable ljg-travel
# 测试

# 同时启用两个
npx skills enable travel-planner
npx skills enable ljg-travel
# 对比效果
```

**解决方案**：

- 根据实际需求选择保留哪个技能
- 或者两个都保留，测试效果后决定
- 大部分情况下，多技能协同工作，不会冲突

## ❓ 常见问题FAQ

### Q1: 安装技能会修改我的代码吗？

**A**: **不会！** 技能通过"知识注入"方式增强系统：
- 代码文件保持原样（`router.js`、`agents/*.js` 等不修改）
- Git 仓库完全干净
- 技能知识在运行时临时注入到 LLM 提示词

### Q2: 卸载技能会影响系统吗？

**A**: **不会！** 随时可卸载：
```bash
# 卸载单个技能
npx skills remove travel-planner

# 卸载所有技能
npx skills uninstall-all
```
卸载后，系统自动恢复到基础功能，无任何副作用。

### Q3: 技能会让 Agent 变慢吗？

**A**: **影响很小**：
- 技能知识通常几百 KB 到几 MB
- LLM 处理时间几乎无感
- 复杂技能（如 ljg-travel，12个Agent并行）可能增加 1-2 秒

### Q4: 可以只启用部分技能吗？

**A**: **可以！** 有两种方式：

**方式1：临时禁用**
```bash
npx skills disable travel-planner
npx skills enable travel-planner  # 重新启用
```

**方式2：卸载不需要的**
```bash
npx skills remove itinerary-optimizer
```

### Q5: 技能兼容不同的 LLM 吗？

**A**: **基本兼容**：
- 技能主要通过提示词增强，不依赖特定 LLM
- 推荐使用 Claude 3.5 Sonnet（效果最佳）
- GPT-4 也可以，但部分技能知识可能需要调整

**A**: **完美兼容！**
